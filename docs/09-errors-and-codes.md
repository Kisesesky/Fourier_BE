# 에러 및 상태 코드

권장 에러 포맷
```json
{
  "error": "BadRequest",
  "message": "Invalid input",
  "details": { "field": "title" }
}
```

상태 코드 가이드
| 코드 | 의미 | 예시 |
| --- | --- | --- |
| 400 | 잘못된 요청 | 필수 필드 누락 |
| 401 | 인증 필요 | 로그인되지 않음 |
| 403 | 권한 없음 | 리소스 접근 불가 |
| 404 | 없음 | 리소스 미존재 |
| 409 | 충돌 | 중복 생성 |
| 422 | 검증 실패 | 포맷 오류 |
| 500 | 서버 오류 | 예외 처리 실패 |

주의사항
- Issues 클라이언트는 에러 시 응답 텍스트를 읽으므로, 메시지는 사람이 읽기 쉬워야 합니다.
